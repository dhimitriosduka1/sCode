Features

Core functionality built for querying clusters with minimal manual terminal interaction.

Active Job Dashboard

Track running, pending, and completing jobs. Visual progress bars display elapsed wall time against requested limits alongside core configurations.

Partition Allocations

View GPU partition resources sorted by utilization. Displays total allocations, node status details, and queue pressure.

Leaderboard & Accounting

Monitor cluster GPU utilization trends across users and accounts. Progress bars highlight overall cluster share per queue.

SBATCH Script Tooltips

Hover partition arguments within SBATCH scripts to fetch live partition capacity. Submit directly via the editor title-bar buttons.

Integrated Log Tailing

Open stderr and stdout log files directly. Resolves relative logs directories and template patterns automatically from sbatch properties.

Batch Operations

Select multiple items to dispatch batch cancel operations. Handles pending array ranges without halting active parent processes.

Setup Guide

Deploy the extension to match your development workflow.

1

Connect via Remote SSH

Connect your local VS Code environment to the remote cluster login node using the official VS Code Remote SSH extension.

2

Install Extension Remote

Search for SLURM Cluster Manager in the Extensions pane. Click Install on SSH: [host] to install it directly on the remote server.

3

Open Workspace Folder

Open a folder containing your SBATCH submit scripts. The Slurm Explorer view will initialize in the sidebar and query Slurm binaries.

1

Check Binary Access

Verify that your local system has direct command-line access to standard Slurm binaries (squeue, sinfo, scontrol).

2

Install Locally

Open the Extensions view, search for SLURM Cluster Manager, and click Install to add it to your local environment.

3

Configure Refresh Settings

Open settings to customize the refresh rates if you are querying highly populated clusters to minimize background network queries.
